내가 생각하는 사용자경험 - User Experience

사용자 경험은 디자인한 사람의 의도와는 다른 방향으로 발생되기도 합니다. 디자이너가 처음 계획할 때부터 사용자의 이용 방식을 원하는 방향대로 잘 의도하면 가장 좋겠지만,
모든 사용자의 사용자경험(UX)을 디자이너가 통제할 수 있는 것은 아닙니다.

디자이너는 사용자 리서치를 통해 기존에 디자인한 결과물로부터 어떤 UX가 발생하는지 알아보게 됩니다. 디자이너가 계획하지 못한 방향이라고 하더라도, 사용자 경험이 이루어지고 있다면
그것 역시 일종의 사용자의 니즈로 이해할 수 있습니다.
디자이너는 이러한 피드백 과정을 통해 기존과는 다른 방향으로 제품과 서비스를 발전시킬 수 있으며,
초기에 예측하지 못했던 새로운 타겟층의 사용자까지 만족시킴으로써 더 나은 제품과 서비스를 제공할 수 있게 됩니다.

1. html5의 가장 큰 특징은 무엇일까요?
이전 버전과 차별화되는 html5의 가장 큰 특징은 시맨틱 태그 사용과 멀티미디어 기능 다양화라고 할 수 있습니다.

태그 구조의 가독성과 목적성을 명확히 하는 시맨틱 태그를 사용하여 더욱 견고하고 구조적인 태그설계가 가능해지며,
눈으로 보기에 구조적인 것뿐 아니라 컴퓨터와 브라우저가 해석하기에 구조적인 설계가 가능해진 부분이 큰 특징입니다.
작성된 html 문서를 컴퓨터가 이전보다 명확하게 분석할 수 있게 되었기 때문에,
브라우저 친화적이고 검색 엔진 접근이 유리하며 효율적인 데이터베이스 구축이 가능하다는 큰 장점을 지닙니다.

또한 폭넓은 멀티미디어 기능 지원으로, 향상된 컴퓨터와 브라우저 성능을 대폭 활용하여
보다 동적이고 생동감 있는 페이지 구현이 가능합니다.
음원과 동영상을 페이지에 직접 삽입함으로써 flash 플러그인 없이도 여러 확장자의 멀티미디어를
이용자에게 제공할 수 있다는 것이 큰 장점입니다.
2. 웹표준, 웹접근성, 시멘틱웹에 대해 설명해주세요.
웹표준이란 구조화되고 정형화된 방식으로 html 문서를 작성하도록 약속된 공식화된 규칙입니다.
자체 브라우저만의 기능을 이용하는 것이 아니라 폭넓은 브라우저와 기기에 대응될 수 있도록
공식화된 일정한 태그와 기능을 사용하여, 범용적인 웹사이트를 구축할 수 있도록 하는 데에 목적이 있습니다.

웹표준을 준수하여 작성된 페이지는 웹접근성 향상과도 일맥상통합니다.
다양한 기기와 환경에서의 웹브라우저 접근성을 향상시킴으로써 폭넓은 이용자에 대응할 수 있으며,
스크린리더 등을 사용하는 정보 취약 계층의 브라우저 접근성을 보장함으로써 웹접근성 향상에 크게 기여할 수 있습니다.
또한 미리 약속된 구조적인 태그, 즉 시맨틱 태그를 사용하여 작성된 웹문서는 브라우저 해석이 용이하여,
검색 엔진 대응과 데이터베이스 구축에 친화적인 시맨틱 웹페이지 작성을 가능하도록 합니다.

CSS와 자바스크립트의 발달로 보이는 것을 위한 복잡한 html 태그 구성의 필요성이 줄어들었기 때문에,
html 문서 본래가 의미하고자 하는 바를 일목요연하게 구조화한 시맨틱 웹페이지를 작성함으로써
문서 자체의 역할과 데이터 제공에 충실할 수 있습니다.
3. 자바스크립트 라이브러리 사용시 가장 큰 장점은?
자바스크립트 라이브러리는 바닐라 자바스크립트의 불편한 부분을 보완할 수 있도록 하는 편리한 도구입니다.

길고 복잡한 자바스크립트 태그와 메소드를 간략화하여 보다 가독성 있는 코드 작성을 가능하도록 하며,
바닐라 자바스크립트만으로는 구현하기 힘든 여러 기능 메소드를 제공함으로써 손쉬운 로직 작성을 가능하게 합니다.

5. CSS 미디어쿼리는 어떤 역할을 수행하는지 답하시오.
CSS 미디어쿼리는 반응형 웹사이트 제작의 가장 기본적인 작동 방식입니다.

브라우저의 가로폭을 인식하여 CSS 효과를 다양하게 적용 가능하도록 하는 것이 미디어쿼리의 역할으로,
HTML의 구조 변화 없이도 CSS만으로 다양한 기기에 최적화된 다양한 화면 구성을 가능하게 하는 것이 장점입니다.

미디어쿼리를 이용하여 다양한 브라우저 사이즈에 대응하는 반응형 웹페이지를 제작함으로써
